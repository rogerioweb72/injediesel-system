## Issues Técnicos Encontrados ⚠️

### 1. Testes antigos (auth.spec.ts) estavam quebrados
- **Problema**: Rotas antigas (`/login`, `/matriz/dashboard`) não existem mais
- **Causa**: Login da matriz mudou para `/appmax`; slug dinâmico `/{toSlug(name)}/dashboard`
- **Status**: ✅ Corrigido — novos testes criados em `01-auth.spec.ts`

### 2. Navegação via `page.goto()` quebrava autenticação
- **Problema**: `page.goto()` força reload completo; Supabase session restore async + AuthGuard redireciona para login antes da sessão estar pronta
- **Causa**: Zustand auth state é ephemeral; Supabase `_getUser` leva ~200ms para restaurar
- **Solução**: Todos os testes agora navegam via `navigateTo()` (click em link da sidebar = client-side routing sem reload)
- **Status**: ✅ Resolvido

### 3. Console errors — `Failed to load resource: 400`
- **Problema**: Duas requisições ao Supabase retornam HTTP 400 durante carregamento do dashboard
- **Impacto**: Não bloqueia funcionalidade (UI carrega normalmente), mas indica queries com problema
- **Ação recomendada**: Investigar quais queries retornam 400 no devtools

### 4. Modal Novo Lançamento — não é Dialog padrão
- **Observação**: O componente `NovoLancamentoModal` é um `div.fixed.inset-0` customizado, não usa `shadcn/ui Dialog` nem `[role="dialog"]`
- **Impacto**: Apenas para testes futuros — seletor correto é `div.fixed`
- **Recomendação**: Considerar migrar para `Dialog` do shadcn/ui para acessibilidade (role, focus trap, ESC key)

### 5. Configurações não usa `[role="tab"]`
- **Observação**: As abas "Usuários" e "Empresa" são `<button>` simples, não Radix Tabs
- **Impacto**: Sem impacto de UX, mas sem suporte a keyboard navigation (setas) esperado de tabs
- **Recomendação**: Migrar para `shadcn/ui Tabs` para acessibilidade correta

### 6. Arquivo ECU — upload só testado no front-end
- **Observação**: O upload para Cloudflare R2 não foi testado end-to-end (requer presigned URL real)
- **Impacto**: Funcionalidade crítica de negócio — upload binário até o R2 não validado neste QA
- **Ação recomendada**: Criar conta de teste no R2 e fazer um upload completo manual
